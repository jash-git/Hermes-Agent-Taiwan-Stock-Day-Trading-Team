# CSV 解析陷阱：T86 三大法人買賣超

## 問題描述

T86 API 回傳的 CSV 檔案中，數字欄位可能包含**千分位逗號**且被**雙引號**包裹。

錯誤範例（手動 `split(',')` 解析會出錯）：
```
證券代號,證券名稱,外陸資買進股數,...
2330,台積電,"12,870","23,902",...
```

若用 `line.split(',')` 解析，`"12,870"` 會被拆成 `"12` 和 `870"`，導致：
- 欄位數量不匹配
- 數字解析錯誤（取得 `12` 而非 `12870`）
- 後續欄位全部偏移

## 正確解法

使用 Python 標準庫 `csv.reader`，它會正確處理引號包裹的欄位：

```python
import csv

with open('twse_fund_20260520.csv', 'r', encoding='utf-8-sig') as f:
    reader = csv.reader(f)
    headers = next(reader)
    for row in reader:
        # row 已經是正確分割的欄位串列
        code = row[0]
        name = row[1]
        buy = int(row[2].replace(',', ''))  # 移除千分位逗號後轉 int
        sell = int(row[3].replace(',', ''))
```

## 實際案例（2026-05-20）

錯誤解析（手動 split）：
```
台積電: 買 12 / 賣 870 / 淨 -858  ← 完全錯誤
```

正確解析（csv.reader）：
```
台積電: 買 12,870 / 賣 23,902 / 淨 -11,031  ← 正確
```

## 受影響的欄位

T86 CSV 中以下欄位可能包含千分位逗號：
- 外陸資買進股數（不含外資自營商）
- 外陸資賣出股數（不含外資自營商）
- 外陸資買賣超股數（不含外資自營商）
- 外資自營商買進股數
- 外資自營商賣出股數
- 外資自營商買賣超股數
- 投信買進股數
- 投信賣出股數
- 投信買賣超股數
- 自營商買賣超股數
- 三大法人買賣超股數

## 教訓

**永遠使用 `csv.reader` 解析 CSV，不要手動 `split(',')`。**
